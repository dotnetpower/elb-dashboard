Linear Pipeline + Return Arc (A2)

A single left‑to‑right pipeline wired through the messages themselves: Producers → Queue → Consumers → Topic. Each rail connects straight to a job particle: producer → job (enqueue) and job → cluster (consume) carry a glowing packet while live. When a job completes, a teal completion packet rides the cluster → Topic rail once and is absorbed at the Topic — it crosses over, then dissolves into the terminus instead of piling up. The Topic stays calm (nothing loops inside it), and the submitter is notified by the separate dashed return arc (Topic → submitter, arrowed). Mock data

Standalone proposal — D3 v7 from CDN, mock data. The return arc is the user-facing narrative ("my job's completion comes back to me"); it is not a claim that submitters hold their own Service Bus subscription — those pucks under the Topic are the real system subscribers (dashboard / audit projectors). The arc is drawn only for terminal jobs so it can never imply a not-yet-finished job is done. If live data drops out the view renders an explicit empty state — never a fake-alive simulation.
