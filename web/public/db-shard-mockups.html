Trade-offs

A — Inline Compact

  • 기존 layout 거의 그대로 — 학습 비용 0
  • 스캔하기 쉬운 한 줄
  • 메타 line이 더 빽빽해짐
  • 상세 정보(메모리 마진 등) 표시 못함 → tooltip에 의존

B — Shard Ladder Strip

  • 한눈에 어떤 N이 준비됐는지 보임
  • 현재 클러스터와의 매칭 highlight로 결정 근거가 시각적
  • warmup 진행 시 박스가 하나씩 채워지는 애니메이션 가능
  • row 높이 +30px → 스크롤 압박
  • 리스트가 길어지면 strip이 반복돼 노이즈

C — Expandable Detail

  • 평소 화면이 깨끗 (마이크로 배지 1개)
  • 필요할 때 정확한 숫자 (RAM %, 마진, 추천 사유) 모두 제공
  • power user 친화적 — table로 비교 가능
  • 기본값에선 sharding이 잘 됐는지 확인하려면 한 번 클릭 필요
  • accordion 상태 관리 필요
