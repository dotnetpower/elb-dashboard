# Tags

Every navigated page in this documentation carries one or more tags from the canon below. Click a tag to jump to every page that uses it. Tags are *not* applied to `docs/features_change/*` change notes — those are indexed by date in the [Change Log](changelog.md) and by release in [Releases](releases/index.md).

## Canon

| Tag | Meaning |
|-----|---------|
| `overview` | Landing / orientation pages a first-time visitor reads. |
| `setup` | Pages a researcher or operator needs to deploy or join a deployment for the first time. |
| `user-guide` | Browser workflow pages a researcher uses day-to-day. |
| `operate` | Operator surface — deployment, architecture, research notes. |
| `architecture` | System-level design references (sidecar topology, identity, network). |
| `infra` | Bicep modules, Container Apps template, AKS / Storage / ACR layout. |
| `auth` | MSAL, managed identity, RBAC. |
| `security` | Security hardening, audit follow-ups, threat model. |
| `blast` | BLAST workflow, results, search-space, NCBI compatibility. |
| `terminal` | Browser terminal sidecar (xterm.js / ttyd / WebSocket). |
| `ui` | Dashboard cards, glassmorphism design tokens, accessibility. |
| `agent` | Pages written for AI coding agents (`copilot/*`). |
| `research` | In-progress research notes that informed design decisions. |
| `contributor` | Conventions and workflows for human contributors. |
| `release` | Per-release notes, change log, version management. |
